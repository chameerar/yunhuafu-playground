Auth0 + Custom Backend

This section demonstrates authentication with Auth0 and calling your custom backend APIs.

  • Auth0: Auth0 account -> domain, Application -> Client ID, [Client Secret], Set of backend services (APIs), Allowed Callback URLs
  • Client submit request to Auth0: If there is no identiy yet or no token, it is not authenticated/authorized -> Auth0 auth URl, Request ID token and access token with desired "audience" or "scope" of required backend APIs
  • Client receives response from Auth0, then use the JWTtoken to get user information and call backend APIs:
    • Claims that Auth0 Use to Authenticate: Sub(subject), Name, Email, Picture, etc
    • Claims that Auth0 Use to Authorize: Audience (backend API should check this, https://api1.example.com), Scope (multiple APIs, "read:messages write:files"), permissions
    • Claim: permissions (Optional â€” for RBAC in Auth0), More granular than scope
      • Only present if:
        • RBAC is enabled
        • "Add permissions in the access token" is checked in Auth0 API settings
